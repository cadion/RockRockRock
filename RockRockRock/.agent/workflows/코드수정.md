# 코드 수정 워크플로우

이 워크플로우는 모든 코드 수정 작업에 적용됩니다.

## 필수 절차

// turbo-all

### Step 1: 프로젝트 가이드 읽기
```
docs/PROJECT_GUIDE.md 파일을 읽어 프로젝트 구조와 파일 목적을 파악합니다.
```

### Step 2: 수정 대상 파일 선별
- PROJECT_GUIDE.md의 File Map을 참고하여 수정이 필요한 파일만 선별합니다.
- 모든 파일을 읽지 마세요. 필요한 파일만 읽으세요.

### Step 3: 코드 수정
- 선별된 파일만 읽고 수정합니다.
- 필요한 경우에만 추가 파일을 읽습니다.

### Step 4: PROJECT_GUIDE.md 업데이트
수정 후 다음 사항을 확인하고 필요시 업데이트합니다:
- 파일 목적이 변경되었으면 File Details 섹션 수정
- 새 파일이 추가되었으면 File Map에 추가
- 새 함수가 추가되었으면 해당 파일의 상세 설명에 추가

### Step 5: Git 커밋
- 항상 PROJECT_GUIDE.md 변경사항을 함께 커밋합니다.
- 새로운 기능 계획이 있으면 docs/plans/에 계획서도 함께 커밋합니다.

## 주의사항

- **토큰 최적화**: 불필요한 파일 읽기를 최소화하세요.
- **가이드 최신화**: 코드 수정 후 반드시 PROJECT_GUIDE.md를 업데이트하세요.
- **플래너 포함**: Git 커밋 시 항상 문서도 함께 업로드하세요.
